# Task ID: 316

**Title:** [FRONTEND] Criar página de boas-vindas pós-primeiro-acesso

**Status:** done

**Dependencies:** 56 ✓, 64 ✓, 315 ✓

**Priority:** medium

**Description:** Desenvolver a página `/boas-vindas` que será exibida uma única vez após o primeiro acesso de um usuário, especificamente depois da troca de senha. A página deve ser acolhedora e motivacional.

**Details:**

1.  **Definição da Rota e Componente**: Criar o arquivo `src/app/(app)/boas-vindas/page.tsx` para a nova página de boas-vindas. Esta rota deve fazer parte do layout principal da aplicação `(app)`.2.  **Lógica de Exibição Única (Frontend)**: No `page.tsx` ou em um componente de layout superior, implementar a lógica para garantir que esta página seja exibida apenas uma vez. Isto deve ser feito verificando um flag no estado da sessão do usuário (e.g., `user.hasSeenWelcomePage` ou `user.isFirstLogin`). Se o flag indicar que a página já foi vista ou que o usuário não está no estado de 'primeiro acesso pós-troca de senha', redirecionar o usuário para o `/dashboard`. A lógica para obter o estado do usuário deve reutilizar os mecanismos de autenticação e sessão já existentes, possivelmente via um `useUser` hook ou dados passados por `serverFetch` se for um RSC.3.  **Busca de Dados**: Utilizar as APIs existentes para buscar:    *   Nome do usuário logado para personalização da mensagem.    *   URL do logotipo da academia (se configurado) com base no `subdominio` da rede, conforme alinhado na Task 64. Acesso a estas informações pode ser feito por um endpoint de perfil de usuário (e.g., `/api/v1/user/profile`) ou via contexto da sessão.4.  **Composição da Interface (UI)**:    *   Centralizar o conteúdo principal da página.    *   Exibir o logotipo da academia de forma proeminente (condicionalmente, se disponível).    *   Mensagem de boas-vindas personalizada, como "Olá, [Nome do Usuário]! Bem-vindo(a) à [Nome da Academia]!".    *   Uma breve explicação sobre as funcionalidades do sistema e seus benefícios.    *   Um botão "Começar Configuração" que, ao ser clicado, deve redirecionar o usuário para a rota `/dashboard` (usando `router.push`).5.  **Integração com Backend (Implícito)**: Embora seja uma tarefa de frontend, a lógica de 'exibição única' pressupõe que o backend tenha um mecanismo para marcar que o usuário já completou o primeiro acesso e/ou visualizou a página de boas-vindas após a troca de senha. Esta flag deve ser atualizada no backend após a conclusão da troca de senha e lida pelo frontend. Consultar a equipe de backend sobre o flag específico (`firstAccessCompleted` ou similar) no perfil do usuário.6.  **Reutilização de Componentes**: Utilizar componentes de UI existentes da biblioteca de design do projeto (botões, tipografia, layout de container) para manter a consistência visual. Exemplos podem ser `Button`, `Typography`, `Container` ou componentes similares encontrados em `src/components/ui/` ou `src/app/(app)/_components/`.

**Test Strategy:**

1.  **Cenário de Primeiro Acesso Pós-Troca de Senha**: Simular um fluxo de usuário que acabou de trocar a senha no primeiro acesso. Verificar se o usuário é automaticamente redirecionado para `/boas-vindas`.2.  **Verificação de Conteúdo**: Na página `/boas-vindas`, verificar se a mensagem de boas-vindas está personalizada com o nome do usuário, se o logotipo da academia é exibido corretamente (se configurado) e se a explicação do sistema está presente.3.  **Funcionalidade do Botão**: Clicar no botão "Começar Configuração" e verificar se o usuário é redirecionado corretamente para `/dashboard`.4.  **Cenário de Acesso Subsequente**: Após completar o fluxo e visualizar a página, deslogar e logar novamente com o mesmo usuário. Verificar se o usuário *não* é redirecionado para `/boas-vindas`, mas sim para a rota padrão pós-login (e.g., `/dashboard`).5.  **Acesso Direto Não Autenticado**: Tentar acessar `/boas-vindas` diretamente via URL sem estar logado. Verificar se o usuário é redirecionado para a página de login.
