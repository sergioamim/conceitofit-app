# Task ID: 314

**Title:** Integrar Checklist de Onboarding no Dashboard

**Status:** done

**Dependencies:** 313 ✓

**Priority:** high

**Description:** Integrar um componente de checklist de onboarding no dashboard principal, visível quando o progresso geral estiver abaixo de 100%, e exibir uma mensagem de parabéns quando completo, controlando sua exibição via localStorage. A exibição e as chamadas à API devem ser condicionalmente desativadas se o tenant já tiver completado o onboarding.

**Details:**

1.  **Definição da API de Status de Onboarding**: Criar ou estender uma função de API em `src/lib/api/onboarding.ts` (se não existir, criar o arquivo) para buscar o status do onboarding do usuário. Assumir um endpoint `GET /api/v1/user/onboarding-status` que retorna um objeto `{ overallProgress: number, checklistItems: Array<{ id: string, label: string, completed: boolean }> }`. Definir os tipos TypeScript correspondentes.2.  **Criação do Hook `useOnboardingStatus`**: Desenvolver um hook `useOnboardingStatus` em `src/hooks/useOnboardingStatus.ts`. Este hook deve utilizar o TanStack Query para buscar o status de onboarding, mas **SOMENTE SE** `tenantContext.onboardingCompleto` for `false`. Ele deverá obter `onboardingCompleto` de um `TenantContext` (assumindo a existência de `src/contexts/TenantContext.tsx` e um hook `useTenantContext`). Se `tenantContext.onboardingCompleto` for `true`, o hook não deve disparar a query e deve retornar um estado indicando onboarding completo sem fazer a requisição ao backend. Configurar `staleTime` para 5 minutos (`5 * 60 * 1000`). Gerenciar estados de carregamento e erro.3.  **Implementação do Componente do Checklist**:Criar um novo componente `OnboardingChecklistCard.tsx` em `src/components/onboarding/` que recebe o status do onboarding e exibe os itens da lista com seus estados (completo/pendente) e o progresso geral.4.  **Implementação do Banner de Parabéns**: Criar um componente `OnboardingCongratsBanner.tsx` em `src/components/onboarding/` para exibir uma mensagem de felicitação quando o onboarding estiver 100% completo. Este banner deve ter um mecanismo para ser descartado e não reexibido na mesma sessão, salvando um flag `dismissedOnboarding` no `localStorage`.5.  **Integração no Dashboard**: Modificar `src/app/(app)/dashboard/page.tsx` para:    a.  Obter o status `onboardingCompleto` do `TenantContext` (via `useTenantContext`).    b.  Utilizar o `useOnboardingStatus` hook.    c.  **Condicionalmente**, se `tenantContext.onboardingCompleto` for `false`:        i.  Renderizar o `OnboardingChecklistCard` no topo do dashboard se `overallProgress < 100%`.        ii. Renderizar o `OnboardingCongratsBanner` se `overallProgress === 100%` E o flag `dismissedOnboarding` NÃO estiver presente no `localStorage`. Após a exibição inicial do banner, o componente deve disparar a gravação de `dismissedOnboarding = true` no `localStorage` para evitar reexibições futuras na mesma sessão (ou até que o localStorage seja limpo).    d.  Se `tenantContext.onboardingCompleto` for `true`, nenhum dos componentes de onboarding deve ser renderizado.6.  **Estilização**: Garantir que os novos componentes sigam o guia de estilo existente e sejam responsivos.

**Test Strategy:**

1.  **Teste do Hook `useOnboardingStatus`**: Criar testes unitários para o hook que verifiquem se:    a.  Ele faz a chamada correta à API quando `tenantContext.onboardingCompleto` é `false`.    b.  Ele NÃO faz chamada à API e retorna um estado de 'completo' (ou similar) quando `tenantContext.onboardingCompleto` é `true`.    c.  Lida com estados de carregamento e erro.    d.  Aplica o `staleTime` configurado.2.  **Teste de Exibição do Checklist**:    a.  Simular um `tenantContext.onboardingCompleto = false` e um status de onboarding com `overallProgress < 100%` e verificar se o `OnboardingChecklistCard` é exibido corretamente no dashboard com os itens de checklist.    b.  Simular um `tenantContext.onboardingCompleto = false` e verificar que o `OnboardingChecklistCard` não é exibido quando `overallProgress === 100%`.    c.  Simular um `tenantContext.onboardingCompleto = true` e verificar que NEM o `OnboardingChecklistCard` NEM o `OnboardingCongratsBanner` são exibidos, e nenhuma requisição de API é feita.3.  **Teste do Banner de Parabéns**:    a.  Simular um status de onboarding com `overallProgress === 100%` (e `tenantContext.onboardingCompleto = false`) e `localStorage` vazio, e verificar se o `OnboardingCongratsBanner` é exibido.    b.  Clicar em um botão de 'fechar' (se houver) ou garantir que o `dismissedOnboarding` seja gravado no `localStorage` após a exibição inicial e que o banner não apareça em recarregamentos subsequentes da página.    c.  Simular `overallProgress === 100%` (e `tenantContext.onboardingCompleto = false`) e `localStorage.getItem('dismissedOnboarding') === 'true'`, e verificar que o banner NÃO é exibido.4.  **Testes de Carregamento e Erro**: Testar os estados de carregamento e erro do hook de onboarding, garantindo que o dashboard reaja apropriadamente (e.g., mostrando spinners ou mensagens de erro).5.  **Testes de UI/UX**: Validar visualmente o layout, responsividade e interatividade dos componentes do checklist e do banner em diferentes tamanhos de tela e dispositivos.
