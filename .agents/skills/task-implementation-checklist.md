## Objetivo
Executar um checklist obrigatório antes de considerar uma task pronta.

## Passos
1. Releia a task e subtasks.
2. Liste os módulos afetados.
3. Verifique se houve alteração fora do escopo.
4. Rode build e testes relevantes.
5. Confirme que não introduziu dependência circular.
6. Resuma riscos, pendências e impacto.
7. Só então sugira marcar a task como done.
